# CRM订单模块 - 增量SQL脚本说明

## 概述

本文档描述了从初始数据库升级到包含完整CRM订单模块功能所需的增量SQL变更。

## 变更内容总览

| 序号 | 变更类型 | 表名 | 变更描述 |
| :--- | :--- | :--- | :--- |
| 1 | 修改字段 | crm_order | status字段类型和注释更新（状态机调整） |
| 2 | 添加字段 | crm_contract | 添加process_instance_id字段（BPM集成） |
| 3 | 更新记录 | system_menu | 订单报表菜单升级为一级菜单 |
| 4 | 新增记录 | system_role_menu | 添加订单报表权限关联 |

## 运行顺序

```

│  1. 修改crm_order.status字段（状态机调整）                    │
│                          ↓                                  │

└─────────────────────────────────────────────────────────────┘
```

## 详细变更说明

### 1. 修改crm_order.status字段

**文件**: `crm_order.sql` 第34行

**变更前**:
```sql
`status` tinyint NOT NULL DEFAULT 1 COMMENT '订单状态（1-待支付 2-已支付 3-已发货 4-已完成 5-已取消）',
```

**变更后**:
```sql
`status` int DEFAULT '0' COMMENT '订单状态：0-草稿，10-待审核，20-审核中，30-审核通过，40-审核不通过，50-已完成，60-已取消',
```

**变更原因**:
- 将原有的5种状态扩展为7种状态
- 新增"草稿"、"待审核"、"审核中"、"审核不通过"状态
- 支持订单审批流程（BPM集成）

**状态映射表**:

| 新状态值 | 状态名称 | 说明 |
| :--- | :--- | :--- |
| 0 | 草稿 | 新建订单未提交 |
| 10 | 待审核 | 订单已提交，等待审核 |
| 20 | 审核中 | BPM流程正在执行 |
| 30 | 审核通过 | 流程审批通过 |
| 40 | 审核不通过 | 流程审批拒绝 |
| 50 | 已完成 | 订单已完成 |
| 60 | 已取消 | 订单已取消 |

### 2. 为crm_contract添加process_instance_id字段

**文件**: 新增

**变更前**: crm_contract表无此字段

**变更后**:
```sql
`process_instance_id` varchar(64) DEFAULT NULL COMMENT '流程实例编号',
```

**变更原因**:
- 支持合同审批流程与BPM集成
- 记录审批流程实例ID，便于追踪和查看审批详情

### 3. 更新订单报表菜单配置

**文件**: `crm_order.sql` 第14行

**变更前**:
```sql
INSERT INTO `system_menu` (`id`, `name`, `permission`, `type`, `sort`, `parent_id`, `path`, `icon`, `component`, ...) 
VALUES (6011, '订单报表', 'crm:order:query', 3, 7, 6004, '', '', '', ...);
```

**变更后**:
```sql
UPDATE `system_menu` 
SET `parent_id` = 2397, `type` = 2, `sort` = 61, `path` = 'order/report', 
    `icon` = 'ep:data-line', `component` = 'crm/order/report/index', `component_name` = NULL 
WHERE `id` = 6011;
```

**变更原因**:
- 将订单报表从订单管理的子菜单升级为CRM模块下的一级菜单
- 便于用户直接访问，提升报表功能的可见性

**菜单结构变化**:

```
CRM模块(2397)                          CRM模块(2397)
├── 订单管理(6004)                      ├── 订单管理(6004)
│   ├── 订单查询(6005)                  │   ├── 订单查询(6005)
│   ├── 订单创建(6006)                  │   ├── 订单创建(6006)
│   ├── ...                            │   ├── ...
│   └── 订单报表(6011)  ← 移除          │   └── 订单审批(6010)
└── ...                                ├── 订单报表(6011)  ← 新增一级菜单
                                       └── ...
```

### 4. 添加订单报表角色菜单关联

**文件**: `crm_order.sql`

**变更前**: 无此记录

**变更后**:
```sql
INSERT INTO `system_role_menu` (`id`, `role_id`, `menu_id`, `creator`, `create_time`, `updater`, `update_time`, `deleted`, `tenant_id`) 
VALUES (6011, 2, 6011, '1', NOW(), '1', NOW(), b'0', 1);
```

**变更原因**:
- 将订单报表权限授予超级管理员角色（role_id=2）
- 确保管理员用户可以访问订单报表页面

## 完整增量SQL

```sql
SET NAMES utf8mb4;
SET FOREIGN_KEY_CHECKS = 0;

-- 1. 修改crm_order表的status字段（状态机调整）
ALTER TABLE `crm_order` MODIFY COLUMN `status` int DEFAULT '0' COMMENT '订单状态：0-草稿，10-待审核，20-审核中，30-审核通过，40-审核不通过，50-已完成，60-已取消';

-- 2. 为crm_contract表添加流程实例字段（BPM集成）
ALTER TABLE `crm_contract` ADD COLUMN `process_instance_id` varchar(64) DEFAULT NULL COMMENT '流程实例编号';

-- 3. 更新订单报表菜单为一级菜单（CRM模块下）
UPDATE `system_menu` 
SET `parent_id` = 2397, `type` = 2, `sort` = 61, `path` = 'order/report', 
    `icon` = 'ep:data-line', `component` = 'crm/order/report/index', `component_name` = NULL 
WHERE `id` = 6011;

-- 4. 添加订单报表的角色菜单关联（超级管理员角色）
INSERT INTO `system_role_menu` (`id`, `role_id`, `menu_id`, `creator`, `create_time`, `updater`, `update_time`, `deleted`, `tenant_id`) 
VALUES (6011, 2, 6011, '1', NOW(), '1', NOW(), b'0', 1);

SET FOREIGN_KEY_CHECKS = 1;
```
