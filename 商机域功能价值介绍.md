# 商机域功能价值介绍

> 仓库分支：`202305566333`
> 适用范围：CRM 商机管理 + 报价管理 + 销售预测 + 销售漏斗 + 商机报表
> 核心定位：把"跟单经验"沉淀为"组织数据资产"，让销售过程可视、可测、可复制

---

## 一、功能总览

商机域围绕"**一条商机从线索到回款的全生命周期**"展开，由 5 个相互协同的子模块组成：

| 子模块 | 核心能力 | 业务角色 |
| --- | --- | --- |
| 商机管理 | 商机档案、阶段流转、跟进预警、产品清单 | 销售一线 |
| 报价管理 | 报价草稿→确认→作废、报价快照、报价历史 | 销售支持 / 销售经理 |
| 销售预测 | 阶段转化率 + 时间衰减的加权预测模型 | 销售总监 / 业务负责人 |
| 销售漏斗 | 各阶段商机数与转化率可视化 | 管理层 |
| 商机状态组 | 可配置的阶段模板与赢单率 | 销售运营 |

## 二、核心功能详解

### 2.1 商机管理 —— 让跟单过程"看得见、留得下"

**功能要点**
- 商机档案：商机名称、客户、**商机来源**（主动开发/客户介绍/网络推广/展会/电话营销/其他）、**竞争对手**、预计成交日期、商机金额、整单折扣、产品清单
- 阶段流转：按"商机状态组"配置的阶段推进，支持阶段变更备注，**不可回退**，避免销售伪造进度
- 跟进预警：**未跟进天数**由后端动态计算（contactLastTime 优先，否则用 createTime），超过 7 天标红提醒
- 多视图：我负责的 / 我参与的 / 下属负责的，支撑销售个人工作台与主管视图
- 结束状态：赢单 / 输单 / 无效，并记录结束原因

**带来的实际收益**

| 痛点 | 价值点 |
| --- | --- |
| 销售离职带走客户 | 商机归属可转移、跟进记录全部沉淀，**客户资产不流失** |
| "这单快成了"无法证伪 | 阶段流转不可回退 + 未跟进天数标红，**避免虚假乐观** |
| 不知道对手是谁 | 竞争对手字段统一沉淀，**为后续竞争策略提供数据基础** |
| 不知道哪条线索转化好 | 商机来源字典化，**可按来源统计转化率，反哺获客决策** |
| 销售主管不知道团队在忙什么 | 下属负责 Tab + 跟进记录，**管理透明化** |

---

### 2.2 报价管理 —— 让报价"可追溯、可审计"

**功能要点**
- **创建报价草稿**：一键从商机的产品清单生成报价草稿，自动计算产品原价合计、折扣、报价总额
- **确认报价**：二次确认后状态变为"已确认"，记录确认人、确认时间，**不可再修改**
- **作废报价**：草稿状态可作废；已确认报价不允许作废，保证审计严肃性
- **报价快照**：确认瞬间生成产品明细快照（`crm_business_quotation_snapshot`），即使后续产品价格变动，**历史报价仍可还原**
- **报价历史**：同一商机可多次报价，形成完整报价轨迹
- **最新已确认报价**：提供 `latest-confirmed` 接口，便于在商机详情页直接展示当前生效报价

**带来的实际收益**

| 痛点 | 价值点 |
| --- | --- |
| 销售私下降价无记录 | 确认即锁定 + 不可修改，**杜绝暗箱操作** |
| 几个月后客户扯皮"当初报价不是这个" | 报价快照完整保留产品、数量、单价、折扣，**有据可查** |
| 同一商机多次报价混乱 | 报价历史列表清晰呈现所有版本与状态，**避免版本错乱** |
| 产品调价后老报价对不上 | 快照机制保证历史报价不被后续价格污染，**审计无死角** |
| 不知道某商机报过几次价 | 报价 Tab 一键查看，**节省翻聊天记录的时间** |

---

### 2.3 销售预测 —— 让业绩预期"有依据、可校准"

**功能要点**

预测算法（数据驱动版）：

```
forecastAmount = Σ ( totalPrice × stageConversionRate × timeWeight )

stageConversionRate = 该阶段历史赢单数 / 该阶段历史结束总数
timeWeight = max(0.3, 1 - daysToDeal/180 × 0.7)   # 半年后权重降至地板价 0.3
```

- **已赢单**：全额计入预测金额
- **进行中**：阶段转化率 × 时间衰减因子加权
- **已流失**：排除
- **同比增长率**：(预测金额 - 去年同期成交金额) / 去年同期成交金额 × 100%
- **KPI 卡片**：商机数 / 商机总金额 / 加权预测金额 / 成交金额 / 平均赢单率
- **明细表**：每条商机的阶段、阶段赢单率、商机金额、加权预测金额、预计成交日期
- **按日期预测**：未来 90 天每日预测金额 vs 去年同期对比，支撑现金流规划

**带来的实际收益**

| 痛点 | 价值点 |
| --- | --- |
| 业绩预测靠拍脑袋 | 算法基于历史转化率 + 时间衰减，**预测有数据支撑** |
| 销售报乐观数，财务无法排产 | 加权预测金额更接近真实落地值，**财务可据此安排资金** |
| 不知道哪个阶段是漏水的 | 阶段转化率直接暴露短板，**针对性改进销售流程** |
| 预计成交日期拖半年还在跟 | 时间衰减因子让远期商机权重自动下降，**避免虚假繁荣** |
| 不知道今年比去年增长多少 | 同比增长率自动计算，**无需手工对账** |
| 远期商机占用预测资源 | 半年后权重降至 0.3，**强制销售推动近期商机** |

### 2.4 销售漏斗 —— 让转化瓶颈"一眼可见"

**功能要点**
- 漏斗图：按阶段展示商机数量
- 转化率：当前阶段商机数 / 上一阶段商机数
- 阶段数据表：阶段名称、商机数、转化率
- 流失商机不计入转化率分母

**带来的实际收益**

| 痛点 | 价值点 |
| --- | --- |
| 不知道哪个阶段掉单最多 | 漏斗图直观呈现"宽窄"差异，**秒级定位瓶颈阶段** |
| 改进方案没数据支撑 | 各阶段转化率可对比，**A/B 测试改进效果可量化** |
| 培训资源不知道投在哪 | 转化率最低阶段即培训重点，**培训 ROI 最大化** |

---

### 2.5 商机状态组 —— 让流程"可配置、可复制"

**功能要点**
- 多阶段模板：如"初步接触→需求确认→方案演示→报价→谈判→成交"
- 每阶段配置赢单率（%），用于销售预测与漏斗分析
- 部门适用范围：不同部门可用不同阶段模板
- 状态组可启停，避免历史数据被破坏

**带来的实际收益**

| 痛点 | 价值点 |
| --- | --- |
| 不同业务线销售流程不同 | 部门级配置，**一套系统支撑多种销售模式** |
| 销售流程变更影响历史数据 | 状态组启停机制，**历史数据不受影响** |
| 赢单率靠经验估 | 状态组配置的赢单率与算法结合，**经验可校准为数据** |

## 三、整合后的整体价值

### 3.1 对销售一线

- **少丢单**：未跟进预警强制关注高价值商机
- **少返工**：报价一键生成、自动算折扣，告别 Excel 反复核算
- **少扯皮**：报价快照 + 确认机制，客户赖账有据可查

### 3.2 对销售管理

- **看得清**：下属负责 Tab + 漏斗图 + 预测明细，团队状态实时掌握
- **管得住**：阶段不可回退 + 确认即锁定，流程合规
- **改得准**：阶段转化率定位瓶颈，培训与流程优化有数据支撑

### 3.3 对业务决策

- **预测稳**：加权预测模型替代拍脑袋，财务排产有依据
- **复盘深**：商机来源、竞争对手、结束原因全维度沉淀，复盘有素材
- **增长明**：同比增长率自动计算，业务趋势一目了然

### 3.4 对组织资产

- **客户资产化**：商机归属可转移，销售离职不带走客户
- **过程数据化**：每一条跟进、每一次报价、每一次阶段流转全部留痕
- **经验可复制**：状态组模板 + 赢单率配置，让销冠的流程成为组织的流程

## 四、技术亮点

| 亮点 | 说明 |
| --- | --- |
| 数据驱动的预测模型 | 不依赖人工设定的赢单率，**从历史结束商机中自动统计阶段转化率** |
| 时间衰减因子 | 远期商机权重自动下降，**避免预测虚高** |
| 报价快照机制 | 确认瞬间冻结产品明细，**保证历史报价可还原** |
| 阶段不可回退 | 业务规则约束在代码层，**杜绝伪造进度** |
| 多租户数据隔离 | tenant_id 全程贯穿，**SaaS 化部署无障碍** |
| 权限粒度细 | 查询/创建/更新/删除/导出/确认/作废 7 类权限点，**角色授权灵活** |
