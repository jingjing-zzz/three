## 三、后端API测试（curl命令）

### 3.1 登录获取Token

```bash
TOKEN=$(curl -s -X POST http://localhost:8080/admin-api/system/auth/login \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1" \
  -d '{"username":"admin","password":"admin123"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['accessToken'])")
echo "Token: $TOKEN"
```

### 3.2 营销活动API测试

```bash
# 1. 创建营销活动
curl -X POST http://localhost:8080/admin-api/crm/marketing/campaign/create \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "name": "测试营销活动",
    "type": 1,
    "status": 0,
    "startTime": 1718400000000,
    "endTime": 1720992000000,
    "targetFilter": "{\"age\": {\"gte\": 18}}",
    "description": "测试活动描述"
  }'

# 2. 查询营销活动列表
curl -X GET "http://localhost:8080/admin-api/crm/marketing/campaign/page?pageNo=1&pageSize=10" \
  -H "tenant-id: 1" \
  -H "Authorization: Bearer $TOKEN"

# 3. 查询单个营销活动（替换为实际ID）
curl -X GET "http://localhost:8080/admin-api/crm/marketing/campaign/get?id=1" \

  -H "Content-Type: application/json" \

    "id": 1,
```

## 六、常见问题排查

### 6.1 API返回401 Unauthorized

**原因**: Token过期或未携带

**解决**: 重新获取Token：
```bash
TOKEN=$(curl -s -X POST http://localhost:8080/admin-api/system/auth/login \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1" \
  -d '{"username":"admin","password":"admin123"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['accessToken'])")
```

### 6.2 API返回403 Forbidden

**原因**: 用户无权限

**解决**: 检查用户角色权限配置，确保admin用户有以下权限：
- crm:marketing-campaign:query/create/update/delete
- crm:marketing-sms-batch:query/create/update/delete
- crm:marketing-email-batch:query/create/update/delete
- crm:marketing-customer-care:query/create/update/delete
- crm:marketing-approval:query/approve/reject/delete
- crm:marketing-send-record:query

### 6.3 API返回500 Internal Server Error

**原因**: 后端代码异常

**解决**: 查看后端日志：
```bash
tail -f /tmp/logs/mitedtsm-server.log
```

### 6.4 页面空白或报错

**原因**: 前端路由或组件问题

**解决**: 打开浏览器开发者工具（F12），查看Console和Network标签页：
- Console: 查看JavaScript错误
- Network: 查看API请求是否正常
