# 销售预测模块说明

> 本文档面向开发者、测试与产品人员，说明销售预测模块的预测口径、算法、适用场景与限制。
> 本模块遵循商机域需求基线中 P0-3「销售预测报表（Demo 级但真实计算）」的要求，
> **明确表示这是「阶段概率加权预测」，不是基于客户历史的机器学习预测。**

---

## 一、预测的本质：预测什么？

### 1.1 预测对象

销售预测模块预测的是**现有商机**的预期成交金额，不是"可能新增的潜在商机"。

具体口径：
- **预测对象**：当前已存在于系统中的商机（`crm_business` 表中 `deleted=0` 的记录）
- **预测维度**：按"预计成交日期"（`deal_time`）区间聚合
- **预测口径**：在该区间内预计成交的商机，按当前所处阶段的概率加权计算预期成交金额

### 1.2 三种商机的处理方式

| 商机状态（end_status） | 业务含义 | 是否计入预测 | 计算方式 |
| --- | --- | --- | --- |
| 进行中（`end_status=0`） | 商机未结束，仍在跟进 | ✅ 计入 | `total_price × 当前阶段 percent / 100 × 时间衰减因子` |
| 已赢单（`end_status=1`） | 商机已成交 | ✅ 计入 | `total_price`（全额计入实际成交额） |
| 已流失（`end_status=2`） | 商机失败，输单 | ❌ 排除 | 不计入预测金额 |

> **重要**：已流失商机（`end_status=2`）一律排除在预测之外，避免虚高预测额。

## 二、预测算法说明

### 2.1 核心算法

```
预测金额（forecastAmount） = Σ 商机 totalPrice × 当前阶段 percent / 100 × 时间衰减因子
```

其中：

1. **阶段概率（percent）**：来自商机阶段配置表 `crm_business_status` 的 `percent` 字段，
   表示该阶段商机的赢单概率。例如：
   - 「需求确认」阶段 percent = 10%
   - 「方案报价」阶段 percent = 30%
   - 「合同审核」阶段 percent = 70%
   - 「成交」阶段 percent = 100%

2. **时间衰减因子（timeWeight）**：根据"预计成交日期"距今的天数动态调整：
   ```
   daysToDeal = (dealTime - 当前日期) 的天数差
   
   if daysToDeal <= 0:        timeWeight = 0.3   # 已过期但未成交，降权到 30%
   if daysToDeal >= 180:      timeWeight = 0.3   # 半年后才成交，降权到 30%
   otherwise:                 timeWeight = 1 - (daysToDeal / 180) × 0.7
   ```
   时间衰减的依据：预计成交日期越远，不确定性越大，预测金额应适当下修。

3. **已赢单商机**：`end_status=1` 时，时间衰减因子 = 1，按全额计入实际成交金额。

### 2.2 同比对比

```
同比增长率 = (预测金额 - 去年同期实际成交金额) / 去年同期实际成交金额 × 100%
```

- **预测金额**：本次查询范围内所有有效商机的加权预测额
- **历史同期成交金额**：去年同期一年内（按预计成交日期区间对应的去年同期）实际赢单商机总金额

### 2.3 算法依据

本算法依据商机域需求基线 P0-3：
> `forecastAmount = sum(商机 totalPrice × 当前阶段 percent / 100)`
> 必须排除已流失商机；成交商机的展示口径要明确（可计入实际成交额，不与预测额重复相加）。

时间衰减因子是本模块在基线算法之上的改进，用于更真实反映远期商机的不确定性。

## 三、预测金额与商机金额的关系

### 3.1 为什么有时预测金额 = 商机金额？

当所有商机都处于「成交」阶段（percent = 100%）且都是已赢单（end_status=1）时：

```
预测金额 = Σ totalPrice × 100% × 1 = Σ totalPrice = 商机总金额
```

**这是当前测试数据的情况**：现有 2 条商机都是已赢单，所以预测金额 = 商机金额 = 25000。

### 3.2 何时预测金额 ≠ 商机金额？

正常情况下，预测金额应该 **小于** 商机金额，因为：

1. **进行中商机**：阶段概率小于 100%，加权后金额降低
2. **已流失商机**：金额被排除（预测金额不包含这部分，但商机金额包含）
3. **远期商机**：时间衰减因子降低预测额

例如：
- 商机金额 100 万，其中 30 万进行中（阶段概率 30%）、50 万已赢单、20 万已流失
- 预测金额 = 50 万 × 100% + 30 万 × 30% = 59 万
- 预测金额 < 商机金额（59 万 < 100 万）

### 3.3 何时预测金额 > 商机金额？

理论上不会出现。预测金额永远不会超过商机总金额。

## 四、图表模式说明

销售预测页面支持 4 种图表模式切换（页面顶部按钮组）：

| 模式 | 显示内容 | 适用场景 |
| --- | --- | --- |
| **对比**（默认） | 同时显示商机金额柱、预测金额柱、历史同期折线 | 综合查看三个指标的对比 |
| **商机金额** | 仅显示商机金额柱状图 | 专注查看原始商机金额分布 |
| **预测金额** | 仅显示预测金额柱状图 | 专注查看加权预测金额分布 |
| **预测差异** | 显示「预测金额 - 商机金额」差异柱（正值绿色，负值红色） | 分析预测金额与商机金额的偏离度 |

## 五、API 接口

### 5.1 接口清单

| 接口 | 路径 | 说明 |
| --- | --- | --- |
| 销售预测汇总 | `GET /admin-api/crm/statistics-forecast/summary` | 返回商机数、商机总金额、预测金额、成交金额、同比变化等汇总指标 |
| 销售预测按日聚合 | `GET /admin-api/crm/statistics-forecast/get-forecast-by-date` | 按预计成交日期聚合的图表数据 |
| 销售预测明细分页 | `GET /admin-api/crm/statistics-forecast/page` | 商机级别的预测明细列表（分页） |

### 5.2 请求参数

通用筛选参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `userIds` | `Long[]` | 否 | 负责人 ID 列表 |
| `dealTimeStart` | `String` | 否 | 预计成交日期开始（`yyyy-MM-dd HH:mm:ss`） |
| `dealTimeEnd` | `String` | 否 | 预计成交日期结束（`yyyy-MM-dd HH:mm:ss`） |
| `pageNo` | `Integer` | 否 | 页码（仅 `/page` 接口） |
| `pageSize` | `Integer` | 否 | 每页条数（仅 `/page` 接口，最大 200） |

### 5.3 汇总接口响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `businessCount` | `Integer` | 活跃商机数（进行中 + 已赢单，排除已流失） |
| `totalAmount` | `BigDecimal` | 商机总金额（进行中 + 已赢单，排除已流失） |
| `forecastAmount` | `BigDecimal` | 加权预测金额 |
| `wonAmount` | `BigDecimal` | 已成交金额 |
| `wonCount` | `Integer` | 已成交商机数 |
| `avgWinRate` | `BigDecimal` | 平均阶段概率 |
| `historyAmount` | `BigDecimal` | 去年同期成交金额 |
| `growthRate` | `BigDecimal` | 同比增长率（百分比，可正可负） |
| `achievementRate` | `BigDecimal` | 达成率（成交金额 / 预测金额） |

## 六、使用限制

1. **不预测潜在商机**：本模块只对现有商机做加权预测，不预测"未来可能新增的潜在商机"
2. **不依赖机器学习**：本模块不基于客户历史数据训练模型，是确定性的阶段概率加权
3. **阶段概率来自配置**：阶段概率（percent）必须由管理员在「CRM 基础配置 - 商机阶段」中正确配置
4. **已流失商机排除**：所有统计和预测都排除 `end_status=2` 的商机
5. **时间衰减是经验值**：时间衰减因子的 180 天阈值和 0.3 下限是经验值，可根据业务调整

## 七、测试场景建议

### 7.1 必测场景

| 场景 | 期望结果 |
| --- | --- |
| 所有商机都是已赢单 | 预测金额 = 商机金额 |
| 包含进行中商机 | 预测金额 < 商机金额 |
| 包含已流失商机 | 已流失商机不计入预测金额和商机金额 |
| 进行中商机阶段升级 | 阶段概率提升，预测金额增加 |
| 远期商机（180 天后成交） | 时间衰减因子 = 0.3，预测金额大幅降低 |
| 同比对比 | 去年同期有成交数据时显示同比增长率 |

### 7.2 边界场景

- 无商机数据：所有指标为 0
- 只有一个商机：预测金额 = 该商机的加权金额
- 所有商机预计成交日期已过期：时间衰减因子全部 = 0.3

## 九、下一步优化方向

1. **多维度分组**：支持按负责人、客户、阶段类型分组聚合
