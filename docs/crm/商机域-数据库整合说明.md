## 二、执行前检查

### 2.1 环境检查

```bash
# 1. 检查 Docker 容器状态
docker ps --filter name=mitedtsm-mysql

# 2. 检查 MySQL 版本
docker exec mitedtsm-mysql mysql -uroot -p<密码> -e "SELECT VERSION();"
```

### 4.5 测试假数据验证

```sql
-- 验证测试数据总量
SELECT COUNT(*) AS total FROM crm_business WHERE creator = 'test-data-script' AND deleted = 0;

-- 验证状态分布
SELECT
  CASE end_status
    WHEN NULL THEN '进行中'
    WHEN 1 THEN '已赢单'
    WHEN 2 THEN '已输单'
  END AS status_name,
  COUNT(*) AS cnt,
  SUM(total_price) AS total_amount
FROM crm_business
WHERE deleted = 0 AND creator = 'test-data-script'
GROUP BY end_status;

-- 验证阶段分布
SELECT s.percent AS stage_percent, COUNT(*) AS cnt, SUM(b.total_price) AS total_amount
FROM crm_business b
JOIN crm_business_status s ON b.status_id = s.id
WHERE b.deleted = 0 AND b.creator = 'test-data-script'
GROUP BY s.id, s.percent
ORDER BY s.percent;

-- 验证预测金额计算
SELECT
  SUM(CASE WHEN b.end_status IS NULL THEN b.total_price * s.percent / 100 ELSE 0 END) AS forecast_in_progress,
  SUM(CASE WHEN b.end_status = 1 THEN b.total_price ELSE 0 END) AS won_amount,
  SUM(CASE WHEN b.end_status IS NULL THEN b.total_price ELSE 0 END) AS in_progress_amount,
  SUM(CASE WHEN b.end_status = 2 THEN b.total_price ELSE 0 END) AS lost_amount
FROM crm_business b
JOIN crm_business_status s ON b.status_id = s.id
WHERE b.deleted = 0 AND b.creator = 'test-data-script';
```

## 八、测试数据说明（new-test-business-data.sql）

### 8.1 数据设计原则

1. **覆盖所有测试场景**：
   - 7 个阶段（10%, 20%, 30%, 50%, 80%, 100%, 0%）
   - 3 种状态：进行中（end_status=NULL）、已赢单（end_status=1）、已输单（end_status=2）
   - 预计成交日期分布：过去 90 天、过去 30 天、未来 30 天、未来 90 天、未来 180 天、去年同期

2. **能体现核心业务**：
   - **销售预测**：进行中商机按阶段概率加权、已赢单全额计入、已输单排除
   - **销售漏斗**：阶段流转（高阶段数 < 低阶段数）
   - **同比对比**：去年同期有 4 条成交商机，用于计算同比增长率
   - **阶段流转**：从需求确认（10%）到合同审核（80%）各阶段都有商机

3. **多维度覆盖**：
   - 6 个客户（id 12-17）
   - 4 个负责人（id 1, 100, 103, 104）
   - 金额从 3.5 万到 580 万不等
   - 部分商机有产品行（12 条产品行）

### 8.2 数据分布统计

| 状态 | 数量 | 总金额 |
| --- | --- | --- |
| 进行中（end_status=NULL） | 20 条 | 3,860,000 元 |
| 已赢单（end_status=1） | 12 条 | 2,521,000 元 |
| 已输单（end_status=2） | 8 条 | 2,188,000 元 |
| **合计** | **40 条** | **8,569,000 元** |

按阶段分布：

### 8.3 预期验证结果

执行测试数据后，销售预测页面应显示：
- **商机总金额**（进行中+已赢单）：6,381,000 元

## 十一、商机综合报表菜单说明（new-crm-statistics-report-menu.sql）

### 11.1 菜单信息

### 11.2 权限复用说明

商机综合报表与销售预测页共用 `crm:statistics-forecast:query` 权限，原因：
1. 两者属于同一业务域（销售统计），权限语义一致
