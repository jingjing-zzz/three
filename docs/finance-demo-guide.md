# 财务域演示指南

## 初始化

先执行结构迁移，再导入演示数据：

```bash
cd /Users/hannn/xtu/实习/project/three/dev
docker compose exec -T mysql mysql -uroot -p1234 mitedtsm_database < ../database/new/new-finance-domain.sql
docker compose exec -T mysql mysql -uroot -p1234 mitedtsm_database < ../database/new/demo-finance-data.sql
```

两个脚本都可以重复执行。导入后重新登录一次，让菜单和权限缓存刷新。

## 建议展示路线

1. 打开“ERP -> 财务管理 -> 财务分析”，展示已审核发票、报销、退款、费用以及收支净额。
2. 打开发票管理，筛选“逾期=是”，展示 `DEMO-FP-002`；点击“逾期检测”说明手工检测和每日定时任务共用同一逻辑。
3. 新增发票，往来单位可直接输入，不依赖客户下拉框；保存后执行审核和反审核。
4. 新增报销，选择申请人和财务审批人。保存后会自动启动 BPM，列表只显示“查看审批”，不能绕过流程直接审核。
5. 使用财务审批人账号进入“工作流程 -> 待办任务”完成审批，再返回报销列表查看“审核通过”。
6. 新增退款并执行“不通过”，返回列表编辑该退款；保存后系统会自动重新发起一条审批流程。
7. 打开收款管理，点击“新增客户”创建客户。需要先有对应客户的已审核销售出库单，才能加入收款明细。
8. 打开 CRM 回款计划，展示逾期标记和“逾期检测”；已生成回款的计划会自动清除逾期标记。

## 演示数据说明

| 编号 | 用途 |
| --- | --- |
| `DEMO-FP-001` | 已审核正常发票 |
| `DEMO-FP-002` | 未审核且已逾期发票 |
| `DEMO-BX-001` | 历史已审批报销，供分析汇总 |
| `DEMO-TK-001` | 历史已审批退款，供分析汇总 |
| `DEMO-FY-001` | 已审核日常费用 |

脚本还会创建“演示基本户”和“星云科技（演示客户）”。BPM 流程不要用 SQL 伪造，报销和退款的流程实例应通过页面新增生成。
