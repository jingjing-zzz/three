# 商机域整合交付说明

## 1. 适用范围

本文用于将商机域功能整合到项目 `/home/renmen/product`。适用内容包括商机字段、阶段与状态完整性、报价与报价快照、销售预测/商机报表菜单及测试数据。

不要以旧的 `database-migration.md` 中嵌入的初始化脚本为准；实际执行顺序以 `docker-compose/init/init-mysql.sh` 为准。

## 3. 全新环境（推荐）

全新环境指 MySQL 数据目录为空。此场景**不要手动逐个执行 SQL**，而是让 MySQL 容器首次启动时自动执行 `init-mysql.sh`。

当前脚本顺序如下：

1. `database/base/quartz.sql`、`database/base/ruoyi-vue-pro.sql` 及其余基础 SQL；
2. 国际化、商品分类、文件上传等通用增量；
3. 字典中文与租户货币修复；
4. 商机字段、商机来源字典、商机状态完整性；
5. 报价表、报价快照、报价快照租户字段修复；
6. 销售预测菜单、商机报表菜单；
7. 测试假数据。

### 启动命令

```bash
cd /home/renmen/product/docker-compose
docker compose up -d
docker compose ps
docker logs -f mitedtsm-mysql
```

日志必须出现以下内容才表示初始化完成：

```text
MySQL initialization completed successfully!
```

若日志在该行之前出现 SQL `ERROR`，当前数据库属于**半初始化状态**，不要继续补跑后续脚本；修复 SQL 后应删除 MySQL 数据卷，再重新初始化。

### 重置 MySQL 和 Redis（仅本地开发环境）

该操作会清空数据库、Redis 缓存和登录会话。执行前确认不需要保留数据。

```bash
cd /home/renmen/product/docker-compose

MYSQL_VOLUME=$(docker inspect mitedtsm-mysql --format '{{range .Mounts}}{{if eq .Destination "/var/lib/mysql"}}{{.Name}}{{end}}{{end}}')
REDIS_VOLUME=$(docker inspect mitedtsm-redis --format '{{range .Mounts}}{{if eq .Destination "/data"}}{{.Name}}{{end}}{{end}}')

docker compose down
docker volume rm "$MYSQL_VOLUME" "$REDIS_VOLUME"
docker compose up -d
docker logs -f mitedtsm-mysql
```

## 4. 已有数据库整合

已有库**不要**删除数据卷，也不要执行 `init-mysql.sh`，先备份数据库，再按需求执行增量 SQL。

### 严禁直接在已有库执行的脚本

`database/new/new-crm-business-quotation.sql` 含 `DROP TABLE` / 重建报价表逻辑，仅用于全新库。已有库需要先由数据库负责人评估数据迁移方案，不能直接运行。

### 常规商机域修复顺序

以下命令不会在终端暴露密码，直接使用 MySQL 容器中的 `MYSQL_ROOT_PASSWORD`、`MYSQL_DATABASE` 环境变量：

```bash
cd /home/renmen/product

run_sql() {
  docker exec -i mitedtsm-mysql sh -lc \
    'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" --default-character-set=utf8mb4 "$MYSQL_DATABASE"' \
    < "database/new/$1"
}

# 先执行状态与快照结构修复
run_sql fix-crm-business-status-integrity.sql
run_sql fix-quotation-snapshot-tenant.sql

# 菜单：先恢复报表菜单，再清理旧预测菜单，最后写入最新版预测菜单
run_sql new-crm-statistics-report-menu.sql
run_sql fix-crm-statistics-forecast-menu-duplicate.sql
run_sql new-crm-statistics-forecast-menu.sql
```

若 `system_tenant.currency_code` 缺失，再执行：

```bash
run_sql fix_currency_code.sql
```

商机字段、报价表和报价快照表是否需要执行，必须先根据目标库表结构确认；不要把“新库脚本”直接当作“已有库补丁”。

## 5. 初始化失败排查

### `Duplicate entry '2012' for key 'system_dict_type.PRIMARY'`

这是 `ruoyi-vue-pro.sql` 与 `crm-2024-09-30.sql` 都写入 CRM 商机来源字典导致的冲突。日志会在执行基础 CRM SQL 时停止，后续 `fix_currency_code.sql` 不会执行，因此可能连带出现 `currency_code` 缺失。

处理方式：先合入针对重复字典的初始化修复，再删除 MySQL 数据卷并从头初始化；不要在半初始化数据库中只补一个 `currency_code` 字段后继续使用。

### `Unknown column 'currency_code' in 'field list'`

说明初始化没有完整执行到 `fix_currency_code.sql`。先检查 MySQL 日志中更早出现的第一条 SQL 错误；解决该错误后重新初始化。已有库可单独执行 `fix_currency_code.sql`。
