# 客户域 (Customer) 整合交付说明

## 1. 交付概述

本文档为 **three 项目** 客户域的整合交付说明，描述客户域在项目中已实现的功能模块、核心实体、业务规则、API接口、前端页面以及与平台基础能力的整合关系。

### 1.1 项目信息

| 项目 | 说明 |
|------|------|
| 项目名称 | three |
| 后端框架 | Spring Boot 3.x + MyBatis-Plus |
| 前端框架 | Vue 3 + TypeScript + Element Plus |
| 客户域后端模块 | `mitedtsm-module-crm` |
| 客户域前端路径 | `Web/src/views/crm/customer/`、`Web/src/views/crm/contact/` |
| 数据库表前缀 | `crm_customer`、`crm_contact`、`crm_customer_limit_config`、`crm_customer_pool_config` |
| 数据库初始化SQL | `database/base/crm-2024-09-30.sql` |
| 分析报表SQL | `database/new/new-crm-customer-analysis-report.sql` (积木报表预建13个数据集) |

> **说明**：项目中还存在 ERP 客户模块（`mitedtsm-module-erp`，表 `erp_customer`），用于采购/销售的对公客户主数据管理，与 CRM 客户域为独立的两个体系。

## 2. 功能子模块

### 2.1 客户管理
- **客户列表**：支持分页查询、多维度筛选（客户名称、行业、等级、来源、状态、负责人等）
  - 三个 Tab 视图：我负责的、我参与的、下属负责的
- **创建客户 / 编辑客户**：基本信息录入（名称、行业、等级、来源、星级、状态、联系方式、地区、详细地址等）
- **客户详情**：基本信息 + 联系人Tab + 商机Tab + 合同Tab + 跟进记录Tab
- **客户导入**：支持 Excel 批量导入，可配置是否更新已存在客户
- **客户导出**：Excel 导出当前筛选结果
- **客户转移**：将客户转移给其他负责人
- **客户锁定/解锁**：锁定后不可编辑或转移
- **客户删除**：逻辑删除

### 2.2 公海管理
- **公海客户池**：未跟进/未成交超期客户自动或手动放入公海
- **公海配置**：
  - 启用/禁用公海机制
  - 未跟进放入公海天数
  - 未成交放入公海天数
  - 提前提醒开关及天数
- **领取客户**：从公海领取客户到自己名下
- **分配客户**：管理员将公海客户分配给指定负责人
- **放入公海**：手动将客户放入公海
- **待进入公海提醒**：列表展示即将进入公海的客户，显示剩余天数

### 2.3 客户限制配置
- **拥有客户上限**：按人员/部门维度设置客户拥有上限
- **锁定客户上限**：按人员/部门维度设置锁定客户上限
- **成交客户计数**：可配置成交客户是否计入拥有客户数量

### 2.4 联系人管理
- **联系人列表**：分页查询，关联客户信息
- **创建联系人 / 编辑联系人**：姓名、手机、电话、邮箱、QQ、微信、性别、职务、是否关键决策人、直属上级、地区、详细地址等
- **联系人详情**
- **联系人关联商机**：支持批量关联/取关商机
- **联系人转移**
- **联系人导出**

### 2.5 客户统计分析
#### 2.5.1 客户总量分析
- **按日期统计**：每日新增客户数、总数趋势
- **按用户统计**：每个负责人的客户总数、新增数

#### 2.5.2 客户跟进分析
- **按日期统计**：每日跟进次数趋势
- **按用户统计**：每个负责人的跟进次数
- **按类型统计**：跟进方式分布（电话/邮件/拜访等）

#### 2.5.3 客户转化率
- **首次合同/回款信息列表**：记录每个客户的首次成交时间、合同金额、回款金额

#### 2.5.4 公海客户分析
- **按日期统计**：每日进入公海/从公海领取的客户数
- **按用户统计**：每个负责人名下进入公海的客户数

#### 2.5.5 客户成交周期
- **按日期统计**：从创建到成交的平均周期
- **按用户统计**：每个负责人的客户平均成交周期
- **按地区统计**：不同地区的客户平均成交周期
- **按产品统计**：不同产品的客户平均成交周期

### 2.6 客户画像分析
- **客户地区分布**：按城市/省份维度统计客户数量
- **客户行业分布**：按行业维度统计客户数量及占比
- **客户级别分布**：按 A/B/C 级别统计客户数量及占比
- **客户来源分布**：按来源渠道统计客户数量及占比

### 2.7 排行榜
- **客户数量榜**：按负责人排名客户拥有数量
- **客户跟进榜**：按负责人排名跟进次数
- **联系人数量榜**：按负责人排名联系人数量

### 2.8 待办工作台
- **待跟进客户列表**：展示分配给我、待跟进的客户
- **待进入公海提醒**：展示即将到期进入公海的客户
- **今日需联系客户列表**：展示今日计划联系的客户

### 2.9 积木报表（预建数据集）

基于积木报表（JimuReport）预建 13 个客户分析 SQL 数据集，可直接在 `/jmreport/list` 中使用：

- 客户概览、客户来源分析、客户等级分布、客户行业分布
- 客户星级分布、客户地区分布、客户月度趋势
- 客户负责人统计、客户成交统计、客户跟进分析
- 客户公海分析、客户活动分析、客户菜单入口

## 4. 业务规则

1. **公海机制**：未跟进超过 X 天或未成交超过 Y 天的客户自动掉入公海，其他人员可领取
2. **公海提醒**：支持提前 N 天提醒负责人，客户即将进入公海
3. **客户转移**：客户归属可在人员间转移
4. **客户锁定**：锁定后的客户不可编辑、不可转移、不可放入公海
5. **客户上限**：可按人员/部门设置拥有客户数上限，防止资源垄断
6. **客户导入查重**：导入时可配置"客户名称"重复时是否更新已存在数据
7. **客户星级**：1-5 星分级，辅助客户价值判断
8. **客户等级**：A（重点客户）、B（普通客户）、C（非优先客户）三级
9. **自动掉入公海定时任务**：`CrmCustomerAutoPutPoolJob` 定时扫描并自动将超期客户放入公海
10. **跟进状态联动**：创建跟进记录时自动更新客户的最后跟进时间和下次联系时间

## 9. 部署说明

### 9.1 数据库初始化

- **CRM 表结构**：执行 `database/base/crm-2024-09-30.sql`，包含 `crm_customer`、`crm_contact`、`crm_customer_limit_config`、`crm_customer_pool_config` 等表的 DDL 语句
- **数据字典**：执行 `Server/sql/mysql/ruoyi-vue-pro.sql`，包含客户行业、等级、来源、星级、状态等字典初始化数据
- **积木报表数据集**：执行 `database/new/new-crm-customer-analysis-report.sql`，预建 13 个客户分析报表 SQL 数据集（客户概览、来源分析、等级分布、行业分布、星级分布、地区分布、月度趋势、负责人统计、成交统计、跟进分析、公海分析、活动分析等），在 `/jmreport/list` 中使用
- 支持 MySQL、Oracle、PostgreSQL、SQL Server、Kingbase、OpenGauss、DM8 等多种数据库

### 9.2 后端服务

- 模块路径: `Server/mitedtsm-module-crm`
- 依赖模块: `mitedtsm-module-system`（用户/部门 API）、`mitedtsm-framework`（基础框架）
- 打包后 JAR 与其他模块共同部署，无独立部署要求

### 9.3 前端资源

- 视图路径: `Web/src/views/crm/customer/`、`Web/src/views/crm/contact/`、`Web/src/views/crm/statistics/`
- API 层路径: `Web/src/api/crm/customer/`、`Web/src/api/crm/contact/`、`Web/src/api/crm/statistics/`
- 国际化路径: `Web/src/locales/zh-CN/crm.ts`、`Web/src/locales/en/crm.ts`、`Web/src/locales/ar/crm.ts`
